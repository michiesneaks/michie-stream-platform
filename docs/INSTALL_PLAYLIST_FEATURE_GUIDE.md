# Playlist feature install guide

This package adds a separate playlist management page, playlist analytics, asset-manager playlist analytics, and the server wiring needed to support the new routes.

## Replace these files

Copy these files into your project and overwrite the existing files at the same paths.

### Front end
- `public/playlists.html`
- `public/asset-manager.html`
- `public/dashboard.html`
- `public/favorites.html`
- `public/Scripts/playlists.js`
- `public/Scripts/dashboard.js`
- `public/Scripts/listen.js`

### Backend routes
- `src/routes/playlists.js`
- `src/routes/favorites.js`
- `src/routes/royalties.js`
- `src/routes/playTokens.js`

### Backend services
- `src/services/playlistService.js`

### Server entry
- `src/server.cjs`

## What stays as-is
- `src/routes/catalog.js`
- `src/services/catalogService.js`
- `src/services/profileService.js`
- `src/services/redisService.js`
- `src/services/ethService.js`
- `src/services/ipfsService.js`
- `src/routes/access.js`
- `src/routes/subscriptions.js`

Those existing files are already compatible with this patch, assuming they match the project structure shown in your file tree.

## Why the server file is included

The mount paths already existed, but this updated `src/server.cjs` keeps the playlist-related routes explicitly documented and preserves the legacy alias:
- `POST /api/create-playlist`

That lets older UI flows keep working while the new UI uses:
- `GET /api/playlists/mine/:wallet`
- `POST /api/playlists`
- `POST /api/playlists/:playlistId/add`
- `POST /api/playlists/:playlistId/remove`
- `POST /api/playlists/:playlistId/reorder`
- `GET /api/playlists/:playlistId/analytics`
- `GET /api/playlists/assets/:wallet/analytics`
- `GET /api/playlist-earnings/:wallet`

## Commands to run

From the project root:

```bash
npm install
```

If you already have dependencies installed, that may be enough. This patch does not add a new npm dependency beyond the modules already visible in your project export.

Then restart the server with your usual command, for example:

```bash
node src/server.cjs
```

If you normally use a watcher:

```bash
npx nodemon src/server.cjs
```

## Recommended deployment order

1. Back up your current files.
2. Replace `src/services/playlistService.js` first.
3. Replace the four route files:
   - `src/routes/playlists.js`
   - `src/routes/favorites.js`
   - `src/routes/royalties.js`
   - `src/routes/playTokens.js`
4. Replace `src/server.cjs`.
5. Replace the front-end files.
6. Restart the server.
7. Hard refresh the browser.

## Smoke test checklist

### Playlist page
- Open `playlists.html`
- Create a playlist
- Add an asset from the listen page
- Remove an asset
- Reorder assets
- Toggle public/private

### Analytics
- Playlist page shows totals and top assets
- Asset manager shows playlist placements, playlist plays, and curator payout generated by your assets
- Dashboard links to `playlists.html`
- Asset manager links to `playlists.html`

### Favorites conversion
- Favorites page can still convert a public favorites collection into a playlist
- Converted playlist appears inside `playlists.html`

### Play accounting
- A completed play with a `playlistId` should increment playlist analytics
- Public playlists only earn curator payout on assets with `supporterRoyaltyEnabled = true`

## Important note about revenue totals

Royalty totals on playlists depend on `grossRoyaltyEth` being included in the play-token request flow. If that field is absent, play counts still work, but revenue will remain `0` until settlement data is supplied.
